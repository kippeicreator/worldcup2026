## 日本語サイトで先に強化する領域

1. ルール、ポジション、フォーメーションなど初心者が繰り返し参照できる常設記事
2. World Cup 2026の日程、開催都市、大会方式など公式確認先を明示できる情報
3. 日本代表とアジアサッカーの公式情報を確認するためのガイド
4. 順位表、移籍、試合日程などニュースを読む前提になる仕組みの解説
5. 女子サッカー、南米、アフリカを含む地域・大会の基礎ガイド

## 3段階の拡張計画

### Phase 1: 日本語記事を20〜30本へ

- 既存12記事の正確性、図解、内部リンクを維持する
- 不足カテゴリから、一次情報を確認できる日本語記事を8〜18本追加する
- 同じ検索意図の記事を量産せず、1ページごとの目的を明確にする

### Phase 2: 実際の需要を深掘り

- Google Search Consoleで表示回数、検索語句、掲載順位を確認する
- 継続的に表示されるカテゴリの補足記事と関連ガイドを優先する
- 更新性が高い記事は、確認日、公式リンク、更新作業の負担も評価する

### Phase 3: 上位5記事の英語版を試験導入

- 日本語版で安定した需要と品質が確認できた上位5記事を候補にする
- `/en/` 配下のURL設計、翻訳レビュー、内部リンクを先に決める
- 日本語版と英語版の対応が維持できる範囲で `hreflang` を試験導入する
- 英語版を増やす前に、検索流入、直帰、更新負荷を確認する

## 英語版を始める条件

- 日本語記事が20〜30本に達し、主要カテゴリの空白が減っている
- Search Consoleで翻訳候補となる上位記事を特定できる
- 固有名詞、ルール、日付を英語でも確認するレビュー手順がある
- `/en/`、canonical、`hreflang`、sitemapを一貫して管理できる
- 日本語版と英語版の更新差分を継続して解消できる

## ライブAPIを検討する条件

- 無料・有料プランの利用規約を確認できる
- スコア、日程、順位などのデータ再配布権を確認できる
- APIキーをブラウザーへ公開しない設計にできる
- GitHub Actionsまたはバックエンドで取得・整形できる
- キャッシュと障害時の静的フォールバックを用意できる
- 継続費用と保守時間を許容できる
- 静的ガイドでは満たせない実際のユーザー需要を確認できる

条件を満たすまでは、公式サイトへの確認導線と保守可能な静的コンテンツを優先します。
